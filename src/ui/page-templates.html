/claim/new

페이지 목적

이 페이지는 요양원 사무직원이 새로운 급여 청구서를 작성하기 위한 페이지입니다.

👤 사용자: 요양원 사무장, 간호사, 행정직원
🎯 목적: 수급자의 일일 서비스 제공 내역을 입력하고 청구서 생성
⏰ 사용 시점: 매일 오전 (전날 서비스 기록) 또는 월말 일괄 작업

제공 기능:

  • 수급자 선택 (드롭다운 또는 검색)
  • 서비스 날짜 선택 (달력 UI)
  • 제공 서비스 체크리스트 (방문요양, 방문목욕, 주야간보호 등)
  • 서비스 시간 입력
  • 특이사항 메모
  • 임시저장 / 제출 버튼
💻 개발 참고: - Form validation 필수 (날짜, 시간 검증) - 자동 저장 기능 (5분마다) - 중복 청구 방지 체크 - 공인인증서 서명 연동 필요

📊 청구 현황 > 반려된 청구

/status/rejected

페이지 목적

이 페이지는 관리자가 심사 반려된 청구건을 확인하고 재청구하기 위한 페이지입니다.

👤 사용자: 요양원 원장, 사무장, 청구 담당자
🎯 목적: 반려 사유 확인 및 수정 후 재청구
⏰ 사용 시점: 심사 결과 통보 후 (보통 매월 15일경)

제공 기능:

  • 반려 청구 목록 (테이블 형태)
  • 반려 사유 상세 보기
  • 반려 사유별 필터링
  • 일괄 수정 기능
  • 재청구 버튼
  • 이의신청 작성
  • 반려 통계 (차트)
💻 개발 참고: - 반려 코드별 자동 수정 제안 - 엑셀 다운로드 기능 - 반려 패턴 분석 AI - 이전 반려 이력 조회

👥 수급자 관리 > 수급자 등록

/beneficiary/add

페이지 목적

이 페이지는 입소 담당자가 새로운 수급자를 시스템에 등록하기 위한 페이지입니다.

👤 사용자: 입소 상담사, 사무직원
🎯 목적: 신규 입소자 정보 등록 및 장기요양등급 확인
⏰ 사용 시점: 새로운 수급자 입소 시

제공 기능:

  • 기본 정보 입력 (성명, 주민번호, 연락처)
  • 장기요양 인정번호 입력
  • 등급 정보 자동 조회
  • 보호자 정보 입력
  • 계약 정보 입력
  • 서비스 이용 계획
  • 필수 서류 체크리스트
💻 개발 참고: - 건보공단 API 연동 (등급 조회) - 주민번호 암호화 저장 - 중복 등록 방지 - OCR 기능 (신분증 스캔)

⚡ 자동화 설정 > 실행 스케줄

/auto/schedule

페이지 목적

이 페이지는 시스템 관리자가 자동 청구 작업 스케줄을 설정하기 위한 페이지입니다.

👤 사용자: 시스템 관리자, IT 담당자
🎯 목적: 반복 작업 자동화 설정으로 업무 효율화
⏰ 사용 시점: 초기 설정 및 운영 중 조정 필요 시

제공 기능:

  • 스케줄 목록 (활성/비활성)
  • 새 스케줄 추가 (Cron 표현식)
  • 실행 시간 설정 (매일/매주/매월)
  • 실행 대상 선택 (전체/특정 수급자)
  • 실행 조건 설정
  • 실패 시 재시도 설정
  • 실행 로그 확인
💻 개발 참고: - Cron job 스케줄러 - 실행 큐 관리 (Redis/Celery) - 동시 실행 방지 - 실패 알림 (Email/SMS)

🔧 시스템 설정 > 공인인증서 등록

/system/certificate

페이지 목적

이 페이지는 기관 대표자가 공인인증서를 등록하고 관리하기 위한 페이지입니다.

👤 사용자: 요양원 원장, 대표자
🎯 목적: 장기요양 포털 자동 로그인을 위한 인증서 등록
⏰ 사용 시점: 초기 설정 및 인증서 갱신 시

제공 기능:

  • 인증서 파일 업로드 (.pfx, .p12)
  • 비밀번호 입력 (암호화 저장)
  • 인증서 정보 확인 (발급기관, 유효기간)
  • 만료 알림 설정
  • 인증서 테스트 (포털 연결 확인)
  • 다중 인증서 관리
  • 인증서 삭제/교체
💻 개발 참고: - 로컬 에이전트 필수 (브라우저 제약) - 인증서 안전 저장 (HSM/암호화) - 자동 갱신 알림 - 인증서 백업 기능

/report/monthly

페이지 목적

이 페이지는 경영진이 월별 청구 현황과 수익을 분석하기 위한 페이지입니다.

👤 사용자: 원장, 사무장, 회계 담당자
🎯 목적: 경영 의사결정을 위한 데이터 분석
⏰ 사용 시점: 월초 (전월 실적 분석) 및 수시

제공 기능:

  • 청구액 추이 차트 (6개월)
  • 승인율 통계
  • 서비스별 청구 분석
  • 수급자별 청구 현황
  • 반려 사유 분석
  • 전년 동기 대비
  • PDF/Excel 다운로드
💻 개발 참고: - Chart.js 또는 D3.js 활용 - 실시간 데이터 업데이트 - 대시보드 커스터마이징 - 정기 리포트 이메일 발송

페이지 목적

👤 사용자: 당직 간호사, 일일 청구 담당자
🎯 목적: 빠른 일괄 청구로 시간 절약
⏰ 사용 시점: 매일 오전 정해진 시간

  • 일괄 청구 실행
  • 실시간 진행률 표시
  • 오류 발생 시 개별 처리
  • 완료 리포트
💻 개발 참고: - 백그라운드 작업 처리 - 프로그레스 바 실시간 업데이트 - 부분 실패 처리 - 자동 재시도 로직

/help/guide

페이지 목적

이 페이지는 모든 사용자가 시스템 사용법을 학습하기 위한 페이지입니다.

👤 사용자: 신규 직원, 모든 사용자
🎯 목적: 시스템 사용법 학습 및 문제 해결
⏰ 사용 시점: 처음 사용 시, 기능 확인 필요 시

제공 기능:

  • 단계별 튜토리얼
  • 기능별 사용 설명서
  • 동영상 가이드
  • FAQ (자주 묻는 질문)
  • 검색 기능
  • 용어 사전
  • 문제 해결 가이드
  • PDF 다운로드
💻 개발 참고: - 검색 엔진 구현 - 동영상 임베드 - 인터랙티브 튜토리얼 - 버전별 업데이트 내역

💡 개발자 참고사항

  • 인증: 모든 페이지는 로그인 필수 (세션 관리)
  • 권한: 사용자 역할별 접근 권한 설정 필요
  • 로깅: 모든 작업은 감사 로그 기록
  • 백업: 중요 작업 전 자동 백업
  • 알림: 중요 이벤트 발생 시 실시간 알림
  • 모바일: 반응형 디자인 적용 (태블릿 지원)
  • 브라우저: Chrome, Edge 최적화
